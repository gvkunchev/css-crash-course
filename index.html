Introduction
CSS = Cascading Style Sheets

A language used to describe how a ML document should be presented.

The language can define different presentation for different viewports.

Mostly used with HTML web pages so this presentation will focus on web applications.
Defining CSS
CSS can be defined in three different ways:
  • Inline <div style="color: red;">This is example.</div>
  • Internal 
    <!DOCTYPE html>
<html>
<head>

<style>
 .some-element {
    border: 1px solid red;
  }
</style>

</head>
<body>
...
  • External 
    <!DOCTYPE html>
<html>
<head>
  <link href="css/general.css" rel="stylesheet" type="text/css" />
</head>
<body>
...
In Decipher's context you can also use LESS files.

LESS gives you extra functionality like defining variables, creating loops and nesting CSS definitions.
Syntax
CSS works by applying properties to elements that match a specific selector.

CSS uses the Sizzle engine for selectors.

The properties and their values are defined by CSS itself. 

SELECTOR {
  PROPERTY: VALUE;
  PROPERTY: VALUE;
}
Selectors
The basic selectors that you can use are the following:
  • All elements 
    * {
  display: block
}
  • Name 
    tagname {
  display: block
}
  • ID 
    #nodeID {
  display: block
}
  • Class 
    .node-class {
  display: block
}
  • Attribute 
    [nodeAttribute=value] {
  display: block
}

    You can also use:
    [attribute]
    [attribute=value]
    [attribute^=value]
    [attribute$=value]
    [attribute*=value]
  • Pseudo 
    a:hover {
  color: red;
}

    Other pseudo selectors are:
    ::after
    ::before
    :checked
    :disabled
    :focus
    :nth-child(n)
    :not(selector)
Selector Combinations
You can combine selectors to achieve complex results.

div.selected
div .selected
div, .selected
div > .selected

The source of the CSS, as well as the selector type and complexity, determines the priority of the property.

You can bring a property higher in the priority list like so: 

div.selected {
  background-color: red !important;
}
Browser Prefixes
In order to achieve compatibility and a way of introducing new features to CSS, web-browser developers came up with prefixes.
These are mostly used for CSS3 properties that are still not finalized officially.
  • Safari: -webkit-
  • Chrome: -webkit-
  • Internet Explorer: -ms-
  • Opera: -o-
  • Firefox: -moz-
  • Android: -webkit-
  • iOS: -webkit-
Example: 
.no-select {
  -webkit-user-select: none;  /* Chrome all / Safari all */
  -moz-user-select: none;     /* Firefox all */
  -ms-user-select: none;      /* IE 10+ */
  user-select: none;          /* Likely future */  
}
Media queries
This is a feature that allows us to define different styling for different devices/vewports/orientation/resolution.

The syntax is as follows: 
@media not|only mediatype and (mediafeature and|or|not mediafeature) {
  CSS-Code;
}


Mostly used to control the look for different screen sizes. For example 
@media screen and (max-width: 768px) {
  div.example {
    display: block;
  }
}

Box model
Each element in HTML is using the following model when rendering.



You can control the way the width/height of an element are parsed using the box-sizing property.

content-box - The margin/padding are NOT taken into account. (default value)
border-box - The margin/padding are taken into account.

Position
Along with display, position is one of the most common properties to adjust the elements' style. The property controls the layout and flow of the elements on the page

The property has the following values:
  • Static
  • Relative
  • Absolute
  • Fixed
  • sticky (Don't use)
Float
The float property allows you to adjust the flow of the page so that the text can wrap around the floated element.

You can float an element to the left or to the right of its container.

Overflow
Whenever the container is smaller than the content, you need to consider the overflow property.
The values are as follows:
  • Visible
    THIS IS OVERFLOWING TEXT.
  • Hidden
    THIS IS OVERFLOWING TEXT.
  • Scroll
    THIS IS OVERFLOWING TEXT.
  • Auto

* Same applies for overflow-x, overflow-y which control the property for each axes separately.
Center alignment
Horizontal alignment is easy:
  • Use text-align: center; on the parent for the inline elements.
  • Use margin: 0 auto; on the element itself for the block elements

Vertical alignment can be tricky.
Consider the following ways:
  • Table
  • Table with table-cell
  • Line-height
  • Padding
Transform
Most of the values for transform are not yet widely supported, but it's worth knowing they exist.

You can safely use scale() which can make some old styles mobile compatible.
